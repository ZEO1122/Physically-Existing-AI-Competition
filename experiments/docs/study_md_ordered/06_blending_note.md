# 블렌딩, Soft Voting, Hard Voting, Stacking 정리

## 개요

여러 모델을 합쳐서 최종 예측을 만드는 방법을 보통 앙상블이라고 한다.
이번 실험에서는 `hybrid`와 `physnet` 예측을 조합할 수 있는지 검토했고,
그 과정에서 가장 직접적으로 사용한 방법은 `soft voting` 성격의 블렌딩이었다.

## 1. 블렌딩이란?

블렌딩은 여러 모델의 예측 확률을 가중 평균해서 최종 확률을 만드는 방법이다.

예시:

- `hybrid`: `unstable_prob = 0.80`
- `physnet`: `unstable_prob = 0.90`

가중치를 아래처럼 주면:

- `hybrid 0.1`
- `physnet 0.9`

최종 예측은:

- `0.1 * 0.80 + 0.9 * 0.90 = 0.89`

즉 핵심은:

- 각 모델의 확률값을
- 정해진 비율로
- 선형 평균하는 것이다

## 2. 왜 블렌딩을 쓰는가?

목적은 모델 간 오답 패턴 차이를 활용하는 것이다.

예를 들어:

- 어떤 샘플은 `hybrid`가 잘 맞추고
- 어떤 샘플은 `physnet`이 더 잘 맞출 수 있다

이때 두 모델이 서로 다른 실수를 한다면,
적절한 비율로 섞었을 때 단일 모델보다 logloss가 더 좋아질 수 있다.

특히 아래 조건에서 효과가 날 가능성이 크다.

- 모델 구조가 다름
- 입력 해석 방식이 다름
- 오답 패턴이 다름

이번 대회에서:

- `hybrid`: 구조/mask/geometry 중심
- `physnet`: motion auxiliary 중심

그래서 블렌딩을 시도할 이유는 충분했다.

## 3. Soft Voting

`soft voting`은 각 모델의 확률을 평균하는 방법이다.

예:

- 모델 A: `0.8`
- 모델 B: `0.9`
- 평균: `0.85`

이 방식은 사실상 가장 기본적인 블렌딩과 같다.

### 장점

- 구현이 매우 단순하다
- 확률 정보를 그대로 보존한다
- logloss 최적화 문제와 잘 맞는다

### 단점

- 약한 모델을 섞으면 성능이 오히려 나빠질 수 있다
- 가중치 선택이 중요하다

### 이번 실험에서의 역할

우리가 `hybrid`와 `physnet`의 OOF 확률을 비율별로 평균해 점수를 본 것은
실질적으로 `soft voting`을 한 것이다.

## 4. Hard Voting

`hard voting`은 확률이 아니라 최종 클래스만 투표하는 방식이다.

예:

- 모델 A: unstable
- 모델 B: stable
- 모델 C: unstable

최종 예측:

- unstable

즉 “누가 더 확신했는가”는 보지 않고,
그냥 클래스 다수결만 본다.

### 장점

- 매우 단순하다
- 직관적이다

### 단점

- 확률 정보가 사라진다
- logloss 같은 확률 기반 평가지표에서는 대체로 불리하다

### 이번 대회에서의 적합성

이번 대회는 확률 제출과 logloss 평가가 핵심이라서,
`hard voting`은 일반적으로 비추천이다.

## 5. Stacking

`stacking`은 여러 모델의 예측을 다시 다른 모델의 입력으로 넣어
메타 모델이 최종 예측을 하게 만드는 방법이다.

예:

- 입력:
  - `hybrid prob`
  - `physnet prob`
  - 추가 메타 정보
- 메타 모델:
  - logistic regression
  - lightgbm
  - 작은 MLP 등

즉 단순 평균이 아니라,
“어떤 상황에서 어느 모델을 더 믿어야 하는가”를 따로 학습하는 방식이다.

### 장점

- 단순 평균보다 더 정교하게 조합 가능
- 모델별 신뢰도를 데이터가 학습함

### 단점

- OOF를 엄격히 써야 한다
- 잘못하면 leakage가 생긴다
- 데이터가 작으면 메타 모델이 쉽게 과적합한다

## 6. 세 방법 비교

| 방법 | 입력 | 장점 | 단점 | 이번 대회 적합성 |
|---|---|---|---|---|
| Soft Voting | 확률 | 단순, 안정적, logloss 친화적 | 약한 모델 섞으면 악화 가능 | 높음 |
| Hard Voting | 클래스 | 구현 쉬움 | 확률 정보 손실 | 낮음 |
| Stacking | 모델 예측 + 메타모델 | 가장 유연 | leakage/과적합 위험 | 중간 |

## 7. 이번 실험에서의 실제 결론

블렌딩 실험 결과:

- `physnet convnext_tiny 384` 단독이 최고
- `hybrid`를 아주 적게 섞어도 OOF logloss가 증가

즉 이번 경우에는:

- 블렌딩 시도 자체는 타당했음
- 하지만 결과적으로는 단독 모델이 더 좋았음
- 현재는 `physnet convnext_tiny 384` 단독 제출이 가장 합리적임

## 8. 현재 기준 권장 순서

1. `physnet convnext_tiny 384` 단독 제출
2. 필요하면 아주 보수적인 soft voting 재검증
3. stacking은 정말 추가 증거가 있을 때만 시도

## 9. 한 줄 요약

- `soft voting`: 확률 평균
- `hard voting`: 클래스 다수결
- `stacking`: 메타 모델이 다시 학습

그리고 이번 실험에서는
`soft voting`으로도 단독 `physnet convnext_tiny 384`를 넘지 못했다.
